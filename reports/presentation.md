## 프로젝트 개요

### 연구주제
- 공공 RFP × 산업뉴스 × 규제 데이터 기반 **산업별 Pain Point 분석** 및
  컨설팅 프로세스(ISP·PI·POC·BPR)별 **제안 전략 자동화 데이터 허브 구축**

### 연구질문
- 나라장터 RFP · 산업 뉴스 · 규제 공시를 통합 분석하면
  **산업별 반복 Pain Point** 를 정량화할 수 있는가?
- 도출된 Pain Point가 **ISP · PI · POC · BPR · PMO** 중 어느 컨설팅
  유형 수요와 연결되는지 자동 분류할 수 있는가?
- **Win Strategy 핵심 포인트** 를 자동 추출해 제안서 작성에 활용할 수 있는가?

### 선정이유
- 보험·금융 IT 기획/PM 실무 경험 기반 관심
- 제안서 작성 시 Pain Point 파악·Win Strategy 수립에 수일 소요되는 비효율
- 데이터 파이프라인으로 자동화 시 제안팀 실무 도구로 즉시 전환 가능

## 데이터 전처리

### 1) 결측치 처리
- 추정가격(`presmptPrce`) 결측 → NaN 유지 (미확정 공고 특성 반영, 대체 X)
- 첨부 URL 빈 문자열 → 누락 처리
- 뉴스 본문 HTML 태그 → 정규식으로 모두 제거

### 2) 데이터 타입 변환
- `bidNtceDt`, `pubDate` → datetime
- `presmptPrce`, `bssamt` → numeric (쉼표 제거 후 변환)

### 3) 데이터 정제
- 중복 제거 (`bidNtceNo` + `bidNtceOrd` 기준)
- 파생 변수 생성 (`year`, `year_month`)
- 산업 태그 (보험 / 금융 / IT / 기타)
- 컨설팅 유형 태그 (ISP / PI / POC / BPR / PMO / 구축 / 기타)
- 첨부 PDF → `pdfplumber`, HWP → `hwp5txt` 로 본문 추출
- 형태소 분석 `kiwipiepy` 명사 추출 (길이 ≥ 2)

### 4) 전처리 전후 비교
- 원본 458,421건 → 중복 제거 후 **420,104건**
- 스키마 66컬럼 혼재 → 핵심 17 + 파생 5 → **22컬럼**
- 분류 없음 → `industry × consulting_type` 태그 완료

## 중간 인사이트

### 발견된 패턴
- 공고 **88% 이상이 "기타" 산업 태그** → 도메인 키워드 사전 확충 필요
- 컨설팅 유형 태그 중 **구축(23K)** 이 압도적, ISP/PI 등 사전형은 소수
- 수요기관 상위 20개가 전체 공고의 약 **6%** 점유
- 추정가격 분포가 로그 스케일에서 정규 근접 → 회귀 전처리 시 `log1p` 적용 타당
- 네이버 뉴스 키워드 중 **"ISP 발주"** 에서 실제 ISP 용역 공고 직접 언급 확인

### 가설
- **H1**. 2023→2025 기간 동안 "AI/데이터/클라우드" 키워드 공고 비중이 증가한다
- **H2**. 금융·보험 업종은 "ISP → 구축" 순으로 시간차를 두고 발주가 이어진다
- **H3**. 공고명에 "혁신·재설계·고도화" 키워드 포함 시 평균 예산이 유의하게 크다
- **H4**. 규제 공시(FSC RSS) 발생 후 N개월 내 관련 발주 건수가 증가한다
